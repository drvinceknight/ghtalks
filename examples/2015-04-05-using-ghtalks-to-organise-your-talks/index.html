ghtalks

Organis and share slide decks with gh-pages.

gh-pages is awesome

So I started putting every talk I gave in a separate repository and sharing via gh-pages.

This became messy

Jekyll is awesome

So I thought about using a jekyll site to organise all my talks.

But...

This needed one extra step that I was not happy with: I needed to write a page that linked to slide deck.

Hopefully, ghtalks is useful

This is where `ghtalks` comes in.

Structure


Talks
├── talks
│   └── 2015-04-05-using-gh-talks
│       └── index.html
│       └── urls.yml
│       └── ...
│   └── archive
│       └── 2014-12-25-auraya
│           └── index.tex
│           └── index.pdf
│           └── images
└── header.html
└── head.html
└── main.css
└── README.md

Running


$ cd Talks
$ gh-talks -s

index.html

  • Will be nicely deployed by gh-pages at github.username.io/Talks.
  • No folder structure required: it finds all directories and files (currently set to hunt down `.pdf` and `html`)
  • `urls.yml` lets you add more info
  • `gh-talks -h` for more info (docs coming soon)
